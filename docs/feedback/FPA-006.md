# FPA-006: State Machine Graph Completeness

**Requirement:** FPA-006 — Shared State Machine Synchronization

**Issue:** The spec defines a shared state machine but does not enumerate the complete state graph. The implementation includes 7 states (Uninitialized, Initializing, Running, Paused, ShuttingDown, Terminated, Error) with specific transition rules, but the spec does not specify:

1. The `Error` state and its transitions (currently: Error → ShuttingDown only)
2. Whether `Paused` is a required state or optional
3. The `Initializing` state as distinct from `Uninitialized`

The `Error` state is referenced in FPA-011 (fault handling) as a consequence of partition failure, but FPA-006 doesn't define it as part of the state machine vocabulary.

**Proposed Clarification:** FPA-006 should include the complete state graph with all valid transitions, or explicitly state that the state vocabulary is domain-specific and defined by the contract crate. The current implementation's graph (documented in state_machine.rs) serves as a reasonable default:

- Uninitialized → Initializing
- Initializing → Running | Error
- Running → Paused | ShuttingDown | Error
- Paused → Running | ShuttingDown
- ShuttingDown → Terminated | Error
- Error → ShuttingDown
- Terminated → (terminal)
