# FPA-004: Bus Trait Object Safety

**Requirement:** FPA-004 says transport mode is selectable without recompilation.

**Issue:** The `Bus` trait has generic methods (`publish<M>`, `subscribe<M>`), making it
not object-safe. The `Compositor` therefore takes `InProcessBus` directly rather than
`dyn Bus`. This prevents runtime transport swapping.

**Proposed Resolution:** Either:
1. Make `Bus` object-safe by using type-erased messages (breaking the typed API)
2. Make `Compositor` generic over `B: Bus` (adds generic parameter to all compositor code)
3. Accept that transport selection is a compile-time choice via generics, not runtime — reinterpret "without recompilation" as "without source changes" (satisfied by generics)

**Recommendation:** Option 3 — make Compositor generic `Compositor<B: Bus>`. This allows
the same source code to work with any transport, satisfying the spirit of FPA-004.
