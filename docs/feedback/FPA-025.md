# FPA-025: Time Semantics Layer Distinction

**Requirement:** FPA-025 — Time-triggered Events

**Issue:** The spec distinguishes wall-clock time (layer 0) from logical/simulation time (layer 1), but the EventEngine API accepts a single `current_time: f64` parameter with no layer awareness. The distinction between wall-clock and logical time is entirely the caller's responsibility — the engine itself is layer-agnostic.

**Observation:** This is architecturally correct — the fractal pattern means the event mechanism is uniform, and the *meaning* of time varies by layer. However, the spec's Verification Expectations assume the event system itself enforces time semantics ("a layer 0 event fires at wall-clock T+5s regardless of time scaling"), when in practice it is the compositor at each layer that must pass the appropriate time value.

**Proposed Clarification:** Clarify that the event engine is time-semantic-agnostic. The compositor (or orchestrator) at each layer is responsible for passing the correct time basis to `evaluate()`:
- Layer 0 compositor passes wall-clock elapsed time
- Layer 1+ compositors pass logical/simulation time from the system clock

This keeps the engine uniform (fractal) while the layer-appropriate time semantics emerge from how each compositor invokes the engine.
