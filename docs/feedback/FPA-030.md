# FPA-030: Diataxis Structure Implementation Gap

**Requirement:** FPA-030 — Documentation Structure (Diataxis)

**Issue:** The spec requires each partition crate to have a `docs/` directory with Diataxis subdirectories (tutorials, how-to, reference, explanation, design). The prototype crates have `docs/` with `SPECIFICATION.md` at the top level, but no Diataxis subdirectories were created.

Additionally, FPA-040 specifies that the contract crate's specification should be at `docs/design/SPECIFICATION.md`, but the prototype places all SPECIFICATION.md files at `docs/SPECIFICATION.md`.

**Proposed Resolution:** For a production implementation:
1. Create the full Diataxis directory structure under each crate's `docs/`
2. Move `SPECIFICATION.md` to `docs/design/SPECIFICATION.md` as specified by FPA-040
3. Populate tutorial, how-to, reference, and explanation directories as content is developed

The prototype defers this structure to keep focus on architectural validation rather than documentation completeness.
