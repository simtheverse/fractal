# FPA-023: Load State Precondition Enforcement

**Requirement:** FPA-023 — State Dump and Load Operations

**Issue:** The spec states "Load shall be invocable only while processing is idle; loading while partitions are actively stepping shall not be supported." The prototype's `Compositor::load()` method does not check execution state — it succeeds regardless of whether the compositor is Running, Paused, or Initializing.

**Observation:** In the prototype's single-threaded lock-step compositor, "loading while partitions are actively stepping" cannot actually occur because `load()` and `run_tick()` cannot execute concurrently. The precondition matters in concurrent/supervisory execution strategies where partitions run on independent tasks.

**Proposed Clarification:** Specify whether "idle" means:
1. The state machine is in a specific state (e.g., Paused or Uninitialized)
2. No partition `step()` calls are in flight (runtime check)
3. Both — state machine must be idle AND no concurrent stepping

For the lock-step compositor, option 1 is sufficient. For the supervisory compositor, option 2 or 3 is needed. The current prototype accepts loads in any state for simplicity.
